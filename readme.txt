程序架构:

|----------------------------------------------|
|			  accept pools					   |	
|accept socket && sticky package && subpackage |
|----------------------------------------------|
				  |
				  | push package
				  V
|-------------------------------------------|					
|			  analyse pools		  			| access database	|-------------------------------------------|
|analyse package && feedback to socket queue|----------------->	|			  dbconnect pools		  		|
|-------------------------------------------|					|-------------------------------------------|

|-------------------------------------------|
|			   send pools		  			|
| 		 flush socket queeue to user		|
|-------------------------------------------|

------------------------------------------------------------------
数据包格式:
	[数据长度,数据]

注意: 
1. 数据长度用字符串(有没有'\0'结束都没有关系)表示，不能超过5个字符

-------------------------------------------------------------------
数据通信协议:

登陆
{
	"type"		: 0,
	"device"	: "999",
	"username"	: "abc", 
	"alias"		: "stevens",			<!-- 可选 -->
	"tags"		: ["home", "school"],	<!-- 可选 -->
}

单发
{
	"type"		: 1,
	"receiver"	: "abc",
	"content"	: "hello"
}


组发
{
	"type"		: 2,
	"tags"		: ["home", "school"],
	"content"	: "hello"
}

群发给所有用户
{
	"type"		: 3,
	"content"	: "hello"
}

群发给所有设备
{
	"type"		: 4,
	"content"	: "hello"
}

消息
{
	"type"		: 1,		<!-- 对应类型的号码, 1为单发、2为组发、3为群发给所有用户、4为群发给所有设备 -->
	"sender"	: "abc",
	"tag"		: "home",	<!-- 只有组发才会有tag字段 -->
	"content"	: "hello"
}
